### 本轮踩坑与修正
- 第一个真实 unified run 的主要问题不是权限，而是 task prompt 太松，Codex 进入 session 后一直在读目录和确认环境，没有在等待窗口内开始写文件。
- 为了解决这个问题，这次把 runner task 改成了更像“作业单”的形式：明确 allowed scope、固定步骤、固定输出文件、固定 heading/schema、固定 state 字段、以及何时算完成。
- 同时把 Codex app-server 的单次等待窗口拉长，避免会话已经开始干活却被 driver 过早判超时。
- 在 `doc-audit` 的 system prompt 和 task 里还修掉了一个真实误报：像 `No archived decisions yet.` / `No recently resolved questions yet.` 这样的空区占位句不应被当成“缺少 Source notes 的 substantive claim”。

### Outcome 验证
