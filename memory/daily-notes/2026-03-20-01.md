## Session 4

### `main.py` handoff enforcement 补强

- 再次复查后确认，仅靠 prompt 和设计文档还不够；如果调度器只要求“有 runtime artifact 变化”，那 Agent 仍然可能只改 `runtime-state.json` 就被视为成功，无法真正保证链路 handoff 完整。
- 因此本轮直接补强了 `agents/main.py`：
  - formal chain agent 被唤醒后，必须真正写出变更后的 downstream work-order JSON
  - 调度器会解析变更后的 work-order JSON，拒绝非法 JSON
  - 调度器会校验最小 handoff 字段：
    - `note-relation` 必须带 `procedural_hints`
    - `doc-maintenance` 必须带 `sop_signals`
    - `sop-promotion` 必须带 promotion result 的核心字段
- 这一步的目的不是让 `main.py` 接管领域判断，而是避免真实链路里出现“Agent prompt 写得对，但运行时悄悄放过坏 handoff”的假成功。
